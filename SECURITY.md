# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest release on main | Yes |
| Older versions | No |

## Reporting a Vulnerability

If you discover a security vulnerability in `wrg-skills` (skill definitions,
audit methodology, disclosure SOP, or this repository's workflows), please
report it responsibly.

**Do not open a public issue.**

Instead, please use GitHub's
[private vulnerability reporting](https://github.com/WRG-11/wrg-skills/security/advisories/new)
or email **detectionfrontier@proton.me**.

### What to include

- Description of the vulnerability
- Steps to reproduce
- Affected surface (skill file / workflow / docs)
- Potential impact
- Suggested fix or mitigation, if any

### Response timeline

- **Acknowledgment:** within 48 hours
- **Initial assessment:** within 1 week
- **Fix (if confirmed):** as soon as practical, typically within 2 weeks

## Scope

This policy covers the `wrg-skills` repository contents (skill definitions,
documentation, CI workflows). It does not cover the upstream MCP servers or
tools that skills invoke — report those vulnerabilities to their respective
maintainers.
